## Architecture

### Summary
Two decoupled subsystems joined by a single frozen contract. A Python 3.12 crawler walks a target tree, dispatches each file to a parser (XML/YAML/JSON/INI), resolves explicit references across files, and emits a deterministic `graph.json`. A static React 18 + Vite viewer loads that `graph.json`, validates it at the boundary, and renders it as a grouped React Flow map with a side panel (param tree + lazy Monaco), fuzzy search, filters, and param-based highlighting. Zero network I/O anywhere. `graph.json` is locked in `docs/GRAPH-SCHEMA.md` as a foundation deliverable before either specialist stream begins (TR3, CLAUDE.md "Key Conventions", WORKFLOW.md L34).

### Data Flow
```
 target-codebase/                 crawler/src/crawler/
  ├─ config/*.xml                 discovery._walk ──► parsers.parse_file
  ├─ pipelines/*.yaml                                      │
  ├─ shared/*.json                                         ▼
  └─ thresholds/*.ini                                FileNode[] + RawRef[]
                                                           │
                                               refs.resolve_references
                                                           │
                                                           ▼
                                              Graph { files[], edges[] }
                                                           │
                                                   emit.to_json (stable sort)
                                                           │
                                                           ▼
                                             ┌────► graph.json ◄────┐
                                             │   (locked contract)  │
                        viewer/public/graph.json        docs/GRAPH-SCHEMA.md
                                             │
                                    lib/graph/load.ts (zod guard, TR5/TR6)
                                             │
                          ┌──────────────────┼──────────────────────────┐
                          ▼                  ▼                          ▼
                graph-store (Zustand)  selection-store           filter-store
                          │                  │                          │
                          ▼                  ▼                          ▼
                GraphCanvas (RF+dagre) FileDetailPanel           FilterBar +
                 FolderGroup | FileNode  ParamTree + Monaco       SearchPalette
                  EdgeStyles (dashed=unresolved)  (lazy)           (Cmd/Ctrl+K)
```

## Non-Goals (v1 — restated from spec, honored ruthlessly)
- **No editing from the viewer.** Read-only Monaco; no writes back to source files.
- **No implicit reference detection.** Only `<include>`, `ref="…"`, YAML `!include`, path imports. No heuristic name matching.
- **No XSD / schema validation** of the target config files.
- **No environment diff / multi-config overlay view.**
- **No full-codebase scale.** One focused module of ~20–50 files; perf work beyond that is out of scope.
- **No network I/O anywhere.** No CDN fonts, no telemetry, no AI, no analytics.
- **No barrel files, no CSS modules, no styled-components** in the viewer.
- **No `print()` in crawler library code** — `logging` only.
- **Source mirroring (`--emit-sources`)** is the minimal mechanism to support the Raw tab; richer history/browsing is out.

## Key Decisions (seed for SUMMARY.md retrospective)
1. **Schema locked via `docs/GRAPH-SCHEMA.md` foundation task** rather than letting it emerge — rationale: biggest identified risk (drift) eliminated, viewer unblocks in parallel.
2. **Committed fixture `graph.json` mirrored in two places** (`crawler/tests/fixtures/sample-module.expected.graph.json` ↔ `viewer/e2e/fixtures/graph.json`) — rationale: lets the two streams run truly parallel; lockstep check in handoff.
3. **Layout engine: dagre for v1**; revisit elkjs only if output is unreadable at 50 nodes.
4. **Self-hosted fonts via `@fontsource/*`** (Inter, JetBrains Mono) — non-negotiable for TR1.
5. **Monaco lazy-loaded** with `React.lazy`, Suspense fallback in panel.
6. **Hide-tests filter default ON** — matches the "new engineer exploring module shape" user story.
7. **Highlighting shows both edge-resolved (strong) and name-match (muted)** — honors spec FR language and avoids hiding ambiguity.
8. **Param value stringification depth-limited to 2**; lists/maps get a `kind` tag and an ellipsis marker.
9. **Node styling: kind-color accent + small badge** — dark-mode scannable.
10. **Windows timestamp convention** uses `HH-MM-SS` (not `HH:MM:SS`) — raise with WORKFLOW.md in `/finalize`.
11. **`--emit-sources` flag** on the crawler is the minimum-surface answer to Raw-source viewing without requiring a backend.

## Notes for Implementer
- **Contract-first is non-negotiable.** If C-stream ever needs to change a field shape, stop and raise a foundation task; both sides update in lockstep (frontend-agent rule from research).
- **Fixture-first TDD in crawler:** write the fixture file + expected parsed params first, then the parser. One parser at a time.
- **Parser isolation:** no parser imports another; lxml types must not escape `parsers/xml.py`.
- **React Flow instance lives in `useReactFlow()`** — not in component state. Selection/filter state in Zustand, layout results memoized.
- **Dark mode primary.** Verify contrast on every new component.
- **Param stringification depth limit = 2.** Anything deeper gets `"…"` and `kind: "map"` or `"list"`. Document the exact truncation marker in `GRAPH-SCHEMA.md`.
- **Tests running on Windows:** don't compare multi-line text without normalizing EOLs; rely on `.gitattributes` for fixtures.
- **Agent routing reminders:**
  - `crawler/` → backend-agent
  - `viewer/` → frontend-agent
  - `docs/`, `.gitignore`, `.gitattributes`, `viewer/public/` regen, fixture copying → execute-agent directly
