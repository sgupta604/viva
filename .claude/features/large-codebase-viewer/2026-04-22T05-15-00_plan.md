# Plan: large-codebase-viewer

**Date:** 2026-04-22T05-15-00 | **Based on:** 2026-04-22T04-49-54_research.md | **Branch:** feat/large-codebase-viewer | **Scope:** XL

## Architecture

### Summary
Land hierarchical cluster rendering + semantic zoom in the viewer (React Flow `parentNode` + elkjs `layered` with `INCLUDE_CHILDREN`, Web-Worker-hosted) and expand crawler ref coverage with three new explicit-pattern edge kinds (XSD, `.d/` aggregation, logical-ID within declared bounds) plus `xi:fallback` classification and optional generated-file detection. `graph.json` version bumps to 2 with a new top-level `clusters[]`, new edge kinds, and two optional `FileNode` flags; viewer retains v1 read compatibility. Default viewer state loads collapsed to depth-1 clusters (auto-descend one level if root has a single child). The folder-filter dropdown's semantics flip from HIDE to NAVIGATE. A 3k-file synthetic Playwright fixture gates FPS at 30 FPS (p95 frame <33 ms) **before** the Viewer stream completes — early-warning measurement so a Cytoscape fallback is viable if it fails.

## Schema Delta

**Canonical contract — JSON camelCase, Python snake_case with `.to_dict()` shim (per xml-viewer-hardening post-finalize lesson).**

### Top-level `Graph`
| Field | v1 | v2 |
|-------|----|----|
| `version` | `1` | `2` |
| `clusters` | — | **NEW** `ClusterNode[]` (always present; empty array if no files) |
| `files`, `edges`, `generatedAt`, `parseErrors` | unchanged | unchanged |

### New type: `ClusterNode`
```ts
{
  path: string;                 // POSIX, relative to crawl root ("base-config/docstorage/global")
  parent: string | null;        // parent cluster path; null for top-level clusters
  childFiles: string[];         // file ids directly contained (not recursive)
  childClusters: string[];      // child cluster paths (not recursive)
  kind: "folder" | "d-aggregate";
}
```

### `FileNode` additions (both optional, default false/null)
| Field | Type | Meaning |
|-------|------|---------|
| `generated` | `boolean` (default `false`) | true only when explicitly listed in a detected templating manifest |
| `generatedFrom` | `string \| null` (default `null`) | POSIX path of manifest that listed this file |

### `Edge` changes
| Field | v1 | v2 |
|-------|----|----|
| `kind` | `"include" \| "ref" \| "import"` | **WIDENED** `"include" \| "ref" \| "import" \| "xsd" \| "d-aggregate" \| "logical-id"` |
| `attrs` | — | **NEW** optional `{ order?: number }` — load order for `.d/` aggregation children |
| `unresolved` | string \| null | **Classification widened:** values may now be prefixed `"fallback:<href>"` (xi:fallback present) or `"ambiguous:<tail>"` (XSD tail collision) |

### Migration story
- Viewer `schema.ts` (zod) accepts both v1 and v2. On v1 input: `clusters = []` (viewer has a safety-net derivation from file paths only if `clusters` is empty or missing — kept minimal, not a full re-implementation), `generated=false`, old edges parsed as-is (no `xsd`/`d-aggregate`/`logical-id` entries possible). No crawler-side v1 emission path. The user's existing fixtures on disk survive a viewer upgrade without a crawler re-run.
- `docs/GRAPH-SCHEMA.md` gains a "Schema Versions" table and a "Key Reference" table (camelCase JSON ↔ snake_case Python) to head off the `parse_error`/`parseError` confusion that bit xml-viewer-hardening post-finalize.

## Non-Goals

- **Heuristic / filename-glob fuzzy ref matching** (no glob-based ID inference; edges must have an explicit anchor). Follow-up feature.
- **Plan Mode** (DECISIONS.md; future feature).
- **Fresh-repo migration** (separate effort per MEMORY.md).
- **XPointer fragment resolution** on XInclude hrefs (deferred).
- **Jinja2 templating introspection** (only mark-as-generated from explicit manifest listing, no variable tracing).
- **Shell scripts and binary files as first-class graph nodes** — they remain string targets in `unresolved` field if referenced, nothing more. Decision per Q8: skip in this feature; revisit as "non-config nodes" follow-up.
- **Feature flag gating.** This IS the new default. Viewer reads v1 graphs via schema shim for backward-load-only, not dual-mode operation.
