# Plan: xml-viewer-hardening

**Date:** 2026-04-21T18-00-00 | **Based on:** `2026-04-21T17-30-00_research.md` | **Branch:** `feat/xml-viewer-hardening`

## Architecture

### Summary
One feature, one branch, one PR — crawler robustness + viewer UX land together on `feat/xml-viewer-hardening`. Crawler side: switch `parsers/xml.py` to lxml recover mode so undeclared `xi:include` and partial-parse cases produce `FileNode`s instead of bare errors; capture `href`/`ENTITY`/path refs; flip `--emit-sources` default ON (adds `--no-emit-sources`); hash-index the O(refs × params) loop in `refs.py`; optionally parallelize parse with a `--jobs` flag, measurement-gated. Viewer side: add a dedicated `view-store` (DECISIONS.md-compliant modular slice) carrying `viewMode ∈ {graph, folders, table}` and `sortBy/sortDir`; ship `FolderView` + `TableView` alongside existing graph; centralize selection through the existing `selection-store`; add a "View raw anyway" affordance in `FileDetailPanel`; reword the `RawSourceView` fallback. Decoupling stays at `graph.json`; no schema change — `parseError` remains a string, partial recovery lands recovered params/refs in the existing arrays.

### Data Flow (what changes)
```

   │                                                        │

   no_network=True)                                     │
   │ ┌──► parser.error_log ──► parse_error (first msg)  ▼
   ▼ │                                    ┌──────┬──────────┬────────┐
 _walk(tree) — normalized _local_name    │graph │ folders  │ table  │
   │       (handles Clark + prefix forms)│ (V1) │  (new)   │ (new)  │
   ▼                                      └──────┴──────────┴────────┘

                                                            ▼
 refs.py: per-source set(param.key) — O(refs)    selection-store → FileDetailPanel
 ─────────                                                │
 crawl() — optional ThreadPoolExecutor                    ▼
 (lxml releases GIL); --jobs N; serial fallback    parse-error → "view raw anyway"
                                                            │
 __main__.py: --emit-sources default ON (add --no-emit-sources)
```

## Schema Delta

**Decision: NO schema change.** Locked from research Open-Q #2.

- `parseError` stays a `string`. First error from `parser.error_log` (or existing exception message) lands there exactly as today.
- Partial-parse params/refs land in the normal `params[]` / `raw_refs[]` arrays, same as a clean parse. UI gets both an error string **and** the recovered content without needing to know "partial vs full".
- `docs/GRAPH-SCHEMA.md` — **no version bump**, but add a one-paragraph note under "FileNode" clarifying: "When `parseError` is non-null, `params` and `raw_refs` MAY still be populated with whatever the recoverable parser extracted." That's the whole doc change.
- `viewer/src/lib/graph/schema.ts` + `types.ts` — **no change.**
- `viewer/e2e/fixtures/graph.json` — **regenerated** as part of F.3 when the sample-module fixture gains `xi-include.xml` and `entity-decl.xml`; byte-locked copy into `crawler/tests/fixtures/sample-module.expected.graph.json`.

## Non-Goals (this feature)

- **No static analysis.** Dead-param detection, orphan-ref detection, cycle detection, cross-file schema inference — all deferred to a follow-up branch (user brief explicit).
- **No Plan Mode.** Architectural-note only via DECISIONS.md TR4; no `planModeStore` scaffolding.
- **No fresh-repo migration.** Stays on existing viva repo; explicitly out of scope per MEMORY.md STATUS note.
- **No `list` view mode.** Research Open-Q #1 resolved: ship **graph + folders + table only**. Table covers the "just let me see them sortable" use case; skipping list saves one component + one spec.
- **No schema struct for `parseError`.** String only (Open-Q #2). Partial-parse surfaces via existing `params`/`raw_refs` arrays.
- **No default parallel parse.** `--jobs=1` default unless measurement shows ≥30 % win on dogfood (Open-Q #3 + Task C.5).
- **No xi:fallback handling.** Out of v1 — raw href captured, fallback children ignored (Open-Q #7).
- **No XPointer fragment resolution** on `href="foo.xml#xpointer(...)"` — treat whole URL as unresolved if file part doesn't match (research Risk §XPointer).
- **No FilterBar refactor.** View state goes in new `view-store.ts`, filter-store.ts stays as is.
- **No docker image change.** `--emit-sources` is a CLI-time flag; no container-build impact (Open-Q #6 verified — runtime image is the viewer, which reads whatever `graph.json` + `source/` the user mounts).

## Key Decisions (resolutions for the seven open questions + seed for SUMMARY.md)

1. **View-mode scope: graph + folders + table (3 modes).** Drop list view — table covers the same use case sortably. Research Open-Q #1.
2. **`parseError` stays a string.** No schema evolution; partial-parse content lands in existing `params`/`raw_refs` arrays; one doc paragraph added to GRAPH-SCHEMA.md. Research Open-Q #2.
3. **Parallel parse opt-in, default `--jobs=1`.** Implemented as ThreadPoolExecutor (lxml releases GIL); measurement-gated flip to default-`cpu_count()` in Task I.2 if ≥30 % dogfood wall-time win. Research Open-Q #3.
4. **View state in dedicated `view-store.ts`**, not in `filter-store.ts`. Respects modular-stores principle (DECISIONS.md TR4); keeps `filter-store` focused on "what to show" vs `view-store` "how to show it". Research Open-Q #4.
5. **"View raw anyway" uses Monaco with `language="plaintext"`** when parseError is set — consistent UX, no highlighting that would choke on partial content. Research Open-Q #5.
6. **Dogfood test is `@pytest.mark.integration` only**, not in default `pytest` run; invoked explicitly in `/test`. Matches existing sample-module pattern. Research Open-Q #6.
7. **ENTITY declarations surface as `RawRef(kind="include")`**; xi:fallback children ignored in v1. Research Open-Q #7.
8. **`recover=True` + `resolve_entities=False` + `no_network=True`** on the lxml parser config — belt-and-suspenders for TR1 (offline). Research Risk §ENTITY-external-DTD.
9. **Foundation sequencing caveat:** F.1 + F.2 run up front; F.3 (fixture regen) deliberately depends on C.2+C.3 landing. Both specialist streams can still start in parallel — V-stream develops against the current committed fixture until F.3 closes, then one fixture swap at the end. No planning change needed; just documented here for the execute-agent.
10. **No changes to docker, deployment, or runtime image.** `--emit-sources` is CLI-time; `source/` sidecar is whatever the user crawls with. Confirmed per Open-Q #6.

## Notes for Implementer

- **Fixture-first TDD on the crawler side** — write `xi-include.xml` + `entity-decl.xml` + failing tests (C.1) before touching `parsers/xml.py` (C.2). This is the v1-demo-retrospective pattern and keeps scope honest.
- **Watch `_local_name` carefully:** under recover mode, a literal `xi:include` tag stays as the string `"xi:include"` — the current Clark-only stripper returns `"xi:include"` unchanged, which would miss the `if tag == "include":` branch. Normalize BOTH `{ns}local` and `prefix:local` (research §Root-cause).
- **Parallel parse determinism:** collect-then-sort by path. Never rely on submission-order == completion-order with a pool. The `test_parallel_parse_determinism` test guards this.
- **Partial-parse testing needs a deliberately-malformed fixture with at least one valid param.** The existing `broken.xml` may or may not have one — check, and if not, author a new `partial.xml` fixture.
- **`BooleanOptionalAction`** requires Python 3.9+ — we're on 3.12, safe.
- **View-store is a fresh file.** Don't extend `filter-store.ts`. The research open-Q #4 landed on separation for modular-stores cleanliness.
- **`groupByFolder` + `sortFiles` compose, don't couple.** The sort happens first (on files), then the group (by folder, preserving row order within folder).
- **Selection-store is already the single source of truth** — FolderView and TableView must call `selectFile(id)`, not implement local state. This is enforced by the V.10 E2E (selection persists across mode switches).
- **Offline safety on entity expansion:** `resolve_entities=False` + `no_network=True` on the parser prevents lxml from fetching external DTDs, which is critical for TR1. Test it via parser-config assertion, not runtime behavior (can't actually block network in CI cleanly).
- **`viewer/dist/` grep** in I.3 — catch any accidental external origin. Self-hosted fonts only (already locked in v1-demo).
- **Commit prefixes:** `fix(crawler):`, `feat(crawler):`, `feat(viewer):`, `test(crawler):`, `test(viewer):`, `docs:`. **No Claude trailers.**
